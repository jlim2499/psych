Which one of the following statements best describes a type I error?

The null hypothesis is rejected when it is true

Type I error = rejecting the null hypothesis when it is true
The correct answer is The null hypothesis is rejected when it is true. In statistical hypothesis testing, a type I error occurs when the null hypothesis (H0) is true, but it's incorrectly rejected. It's equivalent to a false positive result as we are accepting an effect or relationship that doesn't actually exist in the population.

The null hypothesis is accepted when it is false describes a type II error, not a type I error. A type II error occurs when we fail to reject the null hypothesis even though it's false. This means that there really is an effect or relationship in the population, but our study failed to detect it.

The p value fails to reach statistical significance isn't describing any type of error. Rather, this statement refers to a situation where the evidence isn't strong enough to reject the null hypothesis. If our p-value is greater than our significance level (usually 0.05), then we fail to reject H0 and conclude that our data do not provide enough evidence for H1.

The alternative hypothesis is rejected when it is true also describes a Type II error just like 'The null hypothesis is accepted when it is false'. Here, there truly exists a difference or effect in the population but due to limitations of sample size or variability, we fail to detect this difference and incorrectly reject the alternative hypothesis.

A study fails to reach an appropriate power refers to an issue with statistical power, which isn't directly related with type I errors. Statistical power refers to the probability that your test will find a statistically significant result when an effect actually exists. If you have low power, you're less likely to detect an effect if one truly exists - leading potentially towards a Type II error rather than Type I.
